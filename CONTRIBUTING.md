# Contributing (IITA)

## Regla 1: todo va por Issues + PRs
- Crear Issue (bug/feature/test/research/hardware/docs)
- Crear rama
- Abrir PR y linkear Issue

## Regla 2: declarar IA y fuentes
Si se usó IA o fuentes externas, declararlo en el PR y agregar links.

## Regla 3: no subir secretos
Nunca commitear credenciales, tokens, contraseñas. Usar `.env` y GitHub Secrets.

## Regla 4: visibilidad private-first
Los repos de competencia nacen **privados**. No compartir acceso fuera del equipo sin autorización del mentor/responsable. Si el repo se va a hacer público, revisar antes que no haya secretos ni material con derechos de terceros en todo el historial de commits.

## PR checklist mínimo
- Compila / corre
- Evidencia (logs/fotos/videos)
- Docs actualizadas
- Review pedido a responsables (CODEOWNERS)
